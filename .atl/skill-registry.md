## Compact Rules (auto-resolved → inject in sub-agent prompts)

### Brand & Copy (apply to ANY user-facing text or component)

- Spanish copy. Avoid English unless it's a brand term.
- Vertical-agnostic positioning — DO NOT tie copy to a single industry (real-estate-only language was deprecated).
- Black + white + beige palette only. Use Tailwind tokens: `ink`, `cream`, `beige-{50..600}`, `neutral-*`.
- Avoid saturated brand colors.
- Instrument Serif (`font-serif`) is **italic only** — for accents (eyebrow labels with "— " prefix, one decorative word per headline, testimonial quotes).
- Inter (`font-sans`) is the default for body and headlines.
- Logo on dark backgrounds: `brightness-0 invert`.

### Architecture & Code (apply to ANY edit/write of `app/**` or `components/**`)

- Next.js 14 App Router conventions. Server components by default; mark `'use client'` only when needed (state, effects, browser APIs).
- Path alias for imports: `@/components/Foo`, NOT relative paths to `../../`.
- `app/layout.tsx` is the root layout — fonts injected via CSS variables.
- New routes: create `app/{route}/page.tsx`. Co-locate client components in the same folder.
- API routes: `app/api/{name}/route.ts` with named exports `GET`, `POST`, etc.
- TypeScript strict — no `any`, no implicit `any`. Use proper types.
- No test runner configured — do not write `.test.ts` files unless infrastructure is added first.
- Update shared values (waitlist URL, contact info) across ALL components that reference them — they are duplicated by design.

### Build & Lint

- Lint: `npm run lint` (Next.js ESLint).
- Type check: `npx tsc --noEmit`.
- Dev: `npm run dev` on port 3000.
- DO NOT run `npm run build` after changes (per global user rules).

### Commit Style

- Conventional commits only (e.g. `feat:`, `fix:`, `chore:`).
- NEVER add "Co-Authored-By" or AI attribution (per global user rules).
